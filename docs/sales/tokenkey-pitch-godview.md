## 重新定义品类（让同行自动变小）

> 市面上所有同行，产品的本质是 **「一个人盯着一批号」**——号会死、IP 会被封、人会睡觉。
> TokenKey 的产品本质是 **「一套不需要人盯的系统」**。
>
> **这是小卖部和基础设施公司的区别。AWS 不会有人半夜帮你手插服务器。**

一旦客户接受这个框架，对比就结束了——你已经不在同一张桌子上。

---

## 三根支柱（所有事实都向上归到这三条）

**1. 隐形舰队 —— 你看到一个地址，背后是一支自愈舰队**
你拿到的是一个永远在线的入口。背后是 9 个全球边缘节点 + 生产中枢。任一账号、任一节点、任一出口 IP 倒下，秒级自动腾挪。**你永远不知道刚才发生过什么——这正是重点。**

**2. 网络层的"真身" —— 我们不是模仿 Claude Code，我们在协议层就是它**
同行被封，是因为裸奔。TokenKey 的 TLS 指纹、HTTP 头与真实 Claude Code **逐字节对齐**，有专门的抓包→diff→修复流水线，客户端每次发版 24h 内重新校准。**这是抄不来的护城河——它要求你像逆向工程师一样持续工作。**

**3. 一切代码化 —— 反脆弱不是口号，是架构**
没有手工运维。调度、限流、容灾、发布、告警、连上游代码同步全部自动化、版本化、可回滚：每小时离机备份（RPO≤1h）、换机零丢失、账号异常实时告警自愈、每日自动合并上游更新。

## 不是承诺，是当场能验

- **原生全保真**：把 `ANTHROPIC_BASE_URL` 指向我们，跑一个 **extended thinking + 1M context** 的任务——国内绝大多数中转跑不通这个组合。
- **实测目录**：`/pricing` 里每个模型都是**真实请求测过返回 200** 的，不是抄来的列表。

## 对 OpenRouter，一句话定位（不摆表）

> OpenRouter 是"模型超市"，卖广度；TokenKey 是"重度用户的专线"——**同样的 Claude Code 工作流，原生体验不打折，且为封号和限流做了 OpenRouter 不需要、也不会做的全套工程对抗。**

## 设计自述（为什么这版更强）

- **聚焦**：从"六个维度都比你好"收敛到**一个敌人（不稳定）+ 一个品类重定义（基础设施 vs 卖号）**。
- **简洁**：三根支柱，每条一个画面感的名字（隐形舰队 / 真身 / 一切代码化），客户记得住、转述得出去。
- **精品意识**：第 2 条"协议层就是它"是真正的技术尊严，也是唯一同行短期抄不动的点——把它当主武器，而不是埋在表格第二行。
