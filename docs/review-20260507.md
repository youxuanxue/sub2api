## 关键发现

### [R-001] [backend/internal/handler/gateway_handler.go:972] AntigravityModels 响应 shape 破坏性变更
- 严重程度：high
- 问题描述：`PricedCandidates()` 返回 `[]string`，但 `c.JSON` 直接将其序列化为 `data: ["model-id", ...]`；fallback 路径仍返回 `[]antigravity.ClaudeModel`（含 id/type/display_name/created_at）。两条路径的响应 shape 不同，任何解析 `data[]` 为对象的 Claude SDK 或 Antigravity 客户端会在 catalog 非空时收到裸字符串数组而崩溃。
- 符合性参考：docs/approved/pricing-availability-source-of-truth.md §5.2（"响应 shape 完全保留"）
- 建议修复：把 `PricedCandidates()` 输出包装成 `[]antigravity.ClaudeModel`：`ClaudeModel{ID: id, Type: "model", DisplayName: id, CreatedAt: "2024-01-01T00:00:00Z"}`，与 fallback 保持统一 shape。

### [R-002] [backend/internal/service/model_list_filter_tk.go:65] PricedCandidates() 返回全平台 catalog，不是 antigravity-only
- 严重程度：high
- 问题描述：`PricedCandidates()` 调用 `PricedModels()`，后者返回整个 pricing catalog（包含 gpt-4o、gemini-2.5-flash、claude-3-opus 等所有平台）。`AntigravityModels()` 端点语义是"Antigravity 可以服务的模型"，不是全 catalog。客户端若在 `antigravity` group 下选择 `gpt-4o`（来自 pricing catalog），会被调度器拒绝，因为该账号组不能路由 OpenAI-compat 请求。
- 符合性参考：docs/approved/pricing-availability-source-of-truth.md §2.5（"override-default 用 PricedCandidates()"）+ CLAUDE.md §5.x override-default 语义（应保持业务行为不变）
- 建议修复：给 `PricedCandidates(platform string)` 传平台过滤，或对全 catalog 与 `antigravity.DefaultModels()` IDs 取交集（只保留 antigravity catalog 里本来就支持的那些模型），而非整体替换。
