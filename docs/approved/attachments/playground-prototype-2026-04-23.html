Design decisions (Playground v1 — locked at this prototype)

System prompt input?
No (v1). Hidden behind a "v2" hint. Adding system prompts in v1 invites the user to ship production-grade prompts from the playground; we want them to use the API for that.
Show reasoning / o1 thinking output?
No (v1). Upstream schema for thinking blocks (Claude / o1 / Gemini) is still volatile; rendering it in v1 would break every time the upstream changes. Surface only the final assistant message.
Multi-turn context — frontend or server?
Frontend memory only. No server-side conversation entity is created. This keeps blast radius minimal (no DB schema change, no GDPR-style retention question) and matches the "scratchpad" mental model.
Per-call limits
50 turns · 4096 max output tokens · 60s timeout. Documented in the empty-state caption. Anything heavier should use the regular API with the trial key.
Mobile breakpoint?
Desktop-first (≥768px). Mobile layout deferred to follow-up — most early users will explore the Playground from a laptop after creating a key.
