- Stop emitting or forwarding `x-stainless-helper-method`, which is not part of the current real Claude Code request fingerprint we want to mimic.
- Preserve captured real CLI fingerprint values such as `User-Agent` and `X-Stainless-Package-Version` when applying Claude Code mimic defaults, avoiding version downgrade churn on shared OAuth accounts.

## Web / API contract impact
- No frontend page, route, UI state, or i18n surface changed.
- No public API request/response schema changed.
- No admin settings schema changed.

## Why no web change is needed
- This change only affects outbound Anthropic OAuth request headers generated by backend forwarding.
- Existing admin controls for forwarding/mimicry remain unchanged.
- Client-visible payloads and routes are unchanged.
